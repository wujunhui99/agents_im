# PRODUCT_SENSE.md

适用场景：需求存在产品取舍、用户体验边界或“怎么算完成”不清楚时读取本文。

本文档记录产品判断原则，帮助 Agent 在实现时做出一致取舍。

## 核心原则

- 聊天体验优先保证可靠和可理解，再追求复杂能力。
- Agent 行为必须可解释，失败时应给用户明确反馈。
- 群聊中 Agent 不应制造噪音，应尽量响应明确触发或上下文相关请求。
- 工具调用结果需要以用户可理解的方式呈现。

## 用户体验要求

- 用户可见功能必须交付完整产品闭环，而不是只交付后端接口或局部 UI。
- 产品功能进入开发前必须有 GitHub Issue，写清 Product Usability Requirements、Acceptance Criteria 和 Test Plan。
- 图片/文件/媒体类功能只有在发送方可发送、接收方可查看、历史可回显、可预览/下载、失败态可见时才算产品可用。
- 消息状态清晰：发送中、已送达、失败、重试。
- Agent 响应延迟较高时，需要有可感知的状态提示。
- 多 Agent 场景中，需要清晰区分每个 Agent 的身份和职责。
