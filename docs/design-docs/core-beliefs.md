# Core Beliefs

本文档定义本项目的 Agent-first 工程原则。

## 1. 仓库是记录系统

所有重要上下文都应进入仓库，包括需求、设计、执行计划、决策记录、验证结果和技术债。Agent 无法看到的内容，在执行时等同于不存在。

## 2. AGENTS.md 是地图，不是百科全书

`AGENTS.md` 应保持短小，只提供入口和导航。具体规则、背景和决策应沉淀到 `docs/` 下的专题文档中。

## 3. 渐进式披露

Agent 应先读取最小入口，再根据任务读取相关文档，避免一次性加载无关上下文。

## 4. 计划是一等工件

复杂任务必须有执行计划。执行计划应包含任务拆分、验收标准、决策日志、进度和验证结果。

## 5. 可观测性面向 Agent 可读性

日志、指标、链路追踪和错误信息应结构化，便于 Agent 查询、关联和推理。跨服务链路统一使用 `trace_id`。

## 6. 自测和评审循环必须可重复

Generator 完成实现后必须自测；Evaluator 必须检查实现、测试和文档一致性。发现缺陷后进入修复循环，直到满足验收标准。
